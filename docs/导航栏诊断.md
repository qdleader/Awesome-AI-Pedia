# 🔍 导航栏问题排查

如果导航栏没有显示，请按以下步骤排查：

## 1️⃣ 清除浏览器缓存

**Chrome/Edge**:
1. 按 `Ctrl + Shift + Delete` (Windows) 或 `Cmd + Shift + Delete` (Mac)
2. 选择"缓存的图片和文件"
3. 点击"清除数据"

**或者强制刷新**:
- Windows: `Ctrl + Shift + R`
- Mac: `Cmd + Shift + R`

## 2️⃣ 检查部署状态

访问 GitHub Actions 查看部署是否完成：
https://github.com/qdleader/Awesome-AI-Pedia/actions

最新的部署应该显示 ✅ 绿色的勾

**等待时间**: 首次部署可能需要 2-5 分钟

## 3️⃣ 检查页面源代码

1. 访问 https://qdleader.github.io/Awesome-AI-Pedia/
2. 右键点击页面，选择"查看页面源代码"
3. 搜索 `VPNavBar` 或 `nav`
4. 如果找到这些关键词，说明导航栏HTML已生成

## 4️⃣ 检查浏览器控制台

1. 按 `F12` 打开开发者工具
2. 查看 Console 标签是否有错误
3. 查看 Network 标签，看是否有资源加载失败

## 5️⃣ 尝试其他页面

访问以下链接，看导航栏是否显示：
- https://qdleader.github.io/Awesome-AI-Pedia/ai/
- https://qdleader.github.io/Awesome-AI-Pedia/blog/
- https://qdleader.github.io/Awesome-AI-Pedia/guide/

如果这些页面有导航栏，说明只是首页的问题

## 6️⃣ 检查CSS是否被加载

在控制台执行：
```javascript
console.log(getComputedStyle(document.querySelector('.VPNavBar')))
```

## 📸 预期效果

正常情况下，导航栏应该包含：
- **左侧**: Awesome AI Pedia (logo/标题)
- **中间**: 首页 | AI知识库 | 博客 | 指南 | 关于
- **右侧**: GitHub 图标
- **最右侧**: 搜索图标 + 暗黑模式切换按钮

## 🆘 如果仍然无法显示

请提供以下信息：
1. 浏览器类型和版本
2. 浏览器控制台的错误信息（如果有）
3. 访问的完整URL
4. 是否能看到页面其他内容（标题、按钮等）

## ⚡ 快速测试

本地测试：
```bash
npm run dev
```
然后访问 http://localhost:3000/Awesome-AI-Pedia/

如果本地有导航栏，说明配置正确，只是GitHub Pages部署或缓存问题。
